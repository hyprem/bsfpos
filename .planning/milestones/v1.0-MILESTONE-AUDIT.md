### Minor Integration Gaps (P3 — non-blocking)

Both flagged by the integration checker, neither a regression nor a requirement break:

1. **P3-01 — `login-failed` event never emitted by inject.js.** `authFlow.js` reducer accepts `login-failed` in both LOGIN_DETECTED and LOGIN_SUBMITTED states, documented as the "primary signal" from inject.js. `src/inject/inject.js` contains zero `emit('login-failed', …)` calls, and `magiclineView.js` `KNOWN_EVENT_TYPES` whitelist would drop the event even if emitted. **Functional impact: none** — the 8s `POST_SUBMIT_WATCHDOG_MS` fallback in LOGIN_SUBMITTED catches all login failures and transitions to CREDENTIALS_UNAVAILABLE via `loginFailureSideEffects('post-submit-watchdog-expired')`. User experience is correct; only the failure latency is ~8s instead of ~1s. AUTH-04 still fulfilled via the watchdog path.

2. **P3-02 — Admin PIN touch-tap does not round-trip to `idleTimer.bump()`.** Numeric PIN digit keypresses DO bump (via `attachBadgeInput` on host webContents), but touch taps on the PIN modal buttons go through host.js DOM event handlers without pinging the idle timer. Harmless — admin interaction is active by definition and the admin menu is a maintenance surface, not a member-facing one. Strict NFC-05/IDLE-02 compliance for touch-only PIN entry would require wiring a `notify-activity` IPC from host.js.
