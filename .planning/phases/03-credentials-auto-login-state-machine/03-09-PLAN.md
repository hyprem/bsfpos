  <how-to-verify>
    **Probe A — TabTip auto-invoke (under the kiosk user):**
    1. Log in as the Phase 1 Assigned Access kiosk user.
    2. Start the current build of the POS app (installed build preferred; dev build acceptable).
    3. Tap a text input inside the app. Any text input — the credentials overlay from 03-06
       is ideal, otherwise open an ad-hoc `<input type="text">` inside DevTools.
    4. Observe: does the Windows touch keyboard appear automatically?
    5. If NO: from an Electron main-process console (or a temporary `ipcMain.handle` wired into
       a dev build), run:
       ```js
       require('child_process').exec('"C:\\Program Files\\Common Files\\microsoft shared\\ink\\TabTip.exe"');
       ```
       Does the keyboard appear?
    6. If the path is wrong on this box, search: `where /r "C:\Program Files" TabTip.exe`
    7. Record the full path that actually works, or record that nothing works.

    **Probe C — scrypt runtime benchmark:**
    1. On the kiosk, open an Electron main-process console (easiest: a temporary
       `ipcMain.handle('probe-scrypt', ...)` in a dev build, or run `node` from the installer dir).
    2. Run this script and record the output:
       ```js
       const crypto = require('crypto');
       const samples = [];
       for (let i = 0; i < 5; i++) {
         const salt = crypto.randomBytes(16);
         const t0 = process.hrtime.bigint();
         crypto.scryptSync('test-pin-1234', salt, 32, { N: 16384, r: 8, p: 1, maxmem: 64*1024*1024 });
         const t1 = process.hrtime.bigint();
         samples.push(Number(t1 - t0) / 1e6);
       }
       samples.sort((a,b)=>a-b);
       console.log('median ms=', samples[2], 'all=', samples);
       ```
    3. If median is 50-250 ms → keep `N=16384`.
    4. If median < 50 ms → retune to `N=32768`.
    5. If median > 250 ms → retune to `N=8192`.
    6. Also record the CPU model: `wmic cpu get name` in `cmd.exe`.
