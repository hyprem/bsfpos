    **Fastest path (no code, ~2 minutes on the dev box):**

    1. Start any Chromium browser (Electron dev build, Chrome, Edge — anything based on the same
       engine). Open DevTools (F12).
    2. Navigate to `https://bee-strong-fitness.web.magicline.com/#/login`
    3. In the Console, run the following DOM observer:
       ```js
       const log = [];
       const target = document.body;
       const obs = new MutationObserver((muts) => {
         for (const m of muts) {
           for (const n of m.removedNodes) {
             if (n.querySelector && n.querySelector('[data-role="username"]')) {
               log.push('username-removed');
             }
           }
           for (const n of m.addedNodes) {
             if (n.querySelector && n.querySelector('[data-role="username"]')) {
               log.push('username-added');
             }
           }
         }
       });
       obs.observe(target, { childList: true, subtree: true });
       console.log('[probe] observer attached. URL hash is', location.hash);
       ```
    4. Manually type a known-wrong username and password into the form, click submit.
    5. Wait ~3 seconds for Magicline's error response to render.
    6. In the Console, run:
       ```js
       console.log('[probe] log=', log);
       console.log('[probe] hash after submit=', location.hash);
       console.log('[probe] username still mounted?', !!document.querySelector('[data-role="username"]'));
       ```
    7. Record the three values.

    Then write the verdict into
    `.planning/phases/03-credentials-auto-login-state-machine/03-01-KIOSK-VERIFICATION.md`
