<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| host renderer -> main process | admin-menu-action IPC crosses from renderer to main; renderer is untrusted |
| electron-store on disk | posOpen boolean persisted as JSON in user AppData |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-01 | Tampering | toggle-pos-open IPC | mitigate | Existing `if (!adminMenuOpen)` guard at main.js line 759 already rejects admin-menu-action calls when admin menu is not open (requires PIN to enter). No new guard needed — toggle-pos-open inherits this. |
| T-09-02 | Elevation | posOpen in electron-store | accept | posOpen is non-sensitive operational state (not a credential). Tampering with the JSON file on disk requires local Windows user access, which already implies full kiosk control. Low risk. |
| T-09-03 | Denial of Service | Rapid toggle-pos-open spam | accept | Each call is a single synchronous store.set + IPC send. No amplification vector. Already gated behind admin PIN. |
| T-09-04 | Information Disclosure | pos-state-changed IPC payload | accept | Payload contains only `{ posOpen: boolean }`. No PII, no credentials. Broadcast within same BrowserWindow only. |
</threat_model>
