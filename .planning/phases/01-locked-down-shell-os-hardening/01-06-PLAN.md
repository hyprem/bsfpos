**Checkpoint semantics:** This task is human-gated because Phase 1 acceptance is fundamentally a "does it look and feel right" question that cannot be fully automated on a dev machine running prod-sim (see T-06-02 — keyboard chord testing locks the dev out of their own PC). The grep-for-SHELL-IDs automated verify is a *structural sanity check only* — it catches a missing section but cannot detect a fabricated PASS. The load-bearing gate is the human review, not the grep.

Perform the following verification steps in order, capturing output and screenshot paths into `docs/runbook/PHASE-01-ACCEPTANCE.md`. If any step fails, STOP and report back — do not fabricate passes.

**Step 1 — Dev mode smoke test (`npm start`):**
1. From the project directory, run `npm start` in a background process with a 10-second timeout (or foreground + kill).
2. Observe: a 420x800 windowed BrowserWindow should open, showing the Bee Strong dark-background logo, a pulsing yellow loading bar, and "BITTE WARTEN…" text on a `#1A1A1A` background. DevTools should auto-open in detached mode.
3. Capture a screenshot of the window (on Windows: `powershell -Command "Add-Type -AssemblyName System.Windows.Forms; ..."` — if that's too fragile, instruct the user to take a manual screenshot during the human-verify checkpoint at the end).
4. Kill the process.
5. Check `%AppData%/Bee Strong POS/logs/main.log` exists and contains at least one line mentioning `app ready` (electron-log from plan 01 + logger wiring from plan 02/03). Copy the relevant lines into the acceptance doc.

**Step 2 — Single-instance lock test:**
1. Start `npm start` as a background process. Wait 3 seconds for the window to appear.
2. Start a second `npm start` process.
3. Observe: the second process should exit within 1 second with no second window appearing. Verify by checking process count (`tasklist | findstr electron` or equivalent) — there should still be only the original Electron processes.
4. Check `main.log` for the line `second instance detected — exiting silently (D-05)`.
5. Kill the remaining process.

**Step 3 — Prod-simulation smoke test:**
1. Run `npx cross-env NODE_ENV= electron .` to launch without the dev flag (i.e. production behavior on a dev machine). Alternative: `set NODE_ENV= && npx electron .` on Windows cmd, or a PowerShell `$env:NODE_ENV=''; npx electron .`
2. Observe: the window should be fullscreen, frameless, kiosk mode, no menu bar. Splash still visible. DevTools should NOT open.
3. From the keyboard, try: Alt+F4, F11, Escape, Ctrl+W. Observe that none of these close or unfullscreen the window.
4. Try Alt+Tab — the task switcher may briefly appear (this is an OS-level chord that userspace cannot block on its own — the runbook registry hardening handles it on the real device, but not on the developer's unhardened machine). Note this in the acceptance doc as "expected failure on dev machine; mitigated by plan 05 runbook on target device."
5. Try pressing the bare Win key — same note.
6. Exit the app: because Alt+F4 is blocked, use Task Manager (Ctrl+Shift+Esc) or the admin terminal to kill the Electron process. **This is why testing prod mode on a dev machine is awkward and why the checkpoint is human-verify only for dev mode.**

**Step 4 — Keyboard lockdown unit verification (non-interactive):**
Run a quick Node script that imports `keyboardLockdown.js` and exercises `canonical()` for every SHELL-04 required combo, confirming each would be in `SUPPRESS_LIST`. Capture output.

```bash
node -e "
const k = require('./src/main/keyboardLockdown');
const tests = [
  { name: 'Alt+F4',     input: { alt: true, key: 'F4' } },
  { name: 'Alt+Tab',    input: { alt: true, key: 'Tab' } },
  { name: 'F11',        input: { key: 'F11' } },
  { name: 'Escape',     input: { key: 'Escape' } },
  { name: 'Ctrl+W',     input: { control: true, shift: true, key: 'W' } },
  { name: 'Ctrl+w',     input: { control: true, key: 'w' } },
  { name: 'Meta bare',  input: { key: 'Meta' } },
];
let allPass = true;
for (const t of tests) {
  const accel = k.canonical(t.input);
  const suppressed = k.SUPPRESS_LIST.has(accel) || t.input.key === 'Meta';
  console.log(t.name.padEnd(15), '->', accel.padEnd(15), suppressed ? 'SUPPRESSED' : 'LEAKED');
  if (!suppressed) allPass = false;
}
process.exit(allPass ? 0 : 1);
"
```
