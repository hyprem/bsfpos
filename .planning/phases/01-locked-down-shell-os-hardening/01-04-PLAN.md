**Rationale for each nsis key (per RESEARCH.md §Auto-Start Mechanisms and CLAUDE.md):**
- `oneClick: true` — single-click installer, no wizard
- `perMachine: false` — per-user install into `%LocalAppData%\Programs\Bee Strong POS\`, no admin UAC prompt (critical: kiosk account is a standard user, NOT admin)
- `createDesktopShortcut: false` / `createStartMenuShortcut: false` — kiosk has no desktop or start menu; no icons needed
- `runAfterFinish: false` — D-04 safety: the next boot (or `setLoginItemSettings` on relaunch) handles the launch; don't start it from the installer
- `deleteAppDataOnUninstall: false` — preserve logs across reinstalls for post-mortem diagnosis
- `include: build/installer.nsh` — custom NSIS macros for Startup folder shortcut (electron-builder has no first-class option for this)
- `artifactName: ${productName}-Setup-${version}.${ext}` — produces `Bee Strong POS-Setup-0.1.0.exe`

**Create `build/installer.nsh`** with exact content (LF line endings preferred, but NSIS accepts both):

```nsis
# build/installer.nsh
# Custom NSIS macros injected by electron-builder via build.nsis.include.
# Phase 1 plan 04: installs/removes a Startup folder shortcut so the kiosk
# auto-launches on login as one layer of the D-04 belt-and-suspenders pair.
# The runtime layer (app.setLoginItemSettings) is in src/main/main.js.
#
# $SMSTARTUP resolves to the per-user Startup folder
# (%AppData%\Microsoft\Windows\Start Menu\Programs\Startup) when perMachine
# is false, which matches our kiosk account model.
# Ref: electron-userland/electron-builder#1145, electron.build/nsis.html

!macro customInstall
  SetShellVarContext current
  CreateShortCut "$SMSTARTUP\${PRODUCT_NAME}.lnk" "$INSTDIR\${PRODUCT_FILENAME}.exe" "" "$INSTDIR\${PRODUCT_FILENAME}.exe" 0
  DetailPrint "Startup shortcut created: $SMSTARTUP\${PRODUCT_NAME}.lnk"
!macroend

!macro customUnInstall
  SetShellVarContext current
  Delete "$SMSTARTUP\${PRODUCT_NAME}.lnk"
  DetailPrint "Startup shortcut removed: $SMSTARTUP\${PRODUCT_NAME}.lnk"
!macroend
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Installer → user filesystem | NSIS writes to `%LocalAppData%\Programs\Bee Strong POS\` and `$SMSTARTUP`; per-user scope only |
| Installer → Windows user session | No admin elevation (perMachine: false), so the installer cannot tamper with HKLM or system dirs |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | Unsigned installer shown with SmartScreen warning | accept | Code signing is explicitly out of scope per REQUIREMENTS.md. SmartScreen one-click dismiss is acceptable for a single-device install via RDP. Revisit if >1 device is added. |
| T-04-02 | Elevation of Privilege | `perMachine: true` would UAC-prompt | mitigate | Forced `perMachine: false`; install lands in per-user directory; standard-user kiosk account can install without admin |
| T-04-03 | Persistence | Installer does not remove Startup shortcut on uninstall | mitigate | `customUnInstall` macro explicitly Deletes `$SMSTARTUP\${PRODUCT_NAME}.lnk` — uninstall mirrors install |
| T-04-04 | Information Disclosure | Installer bundle includes source maps or planning docs | mitigate | `files` allowlist restricts to `src/**/*` + `package.json`; `!**/*.md` excludes planning artifacts (including CLAUDE.md, .planning/**) |
| T-04-05 | DoS | Installer auto-runs app (`runAfterFinish: true`) producing competing launch with Startup shortcut on first boot | mitigate | `runAfterFinish: false` explicitly; the first launch happens naturally at next user login |
</threat_model>
