## Spacing Scale

Declared values (must be multiples of 4). Confirmed against host.css patterns.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, gap within `.bsk-admin-diagnostics` rows |
| sm | 8px | Gap between admin buttons (`.bsk-admin-btns gap: 8px`), diag row gaps |
| md | 16px | Default element spacing, field gaps, card inner gap units |
| lg | 24px | Logo margin-bottom, section breaks within cards |
| xl | 32px | Card padding (`.bsk-card--admin padding: 32px`), page-level layout padding |
| 2xl | 48px | Major section breaks (not used in admin layer currently) |
| 3xl | 64px | Page-level spacing (not used in admin layer) |

Exceptions:
- Close button tap target: **44x44px minimum** (ADMIN-01, D-01). This is an ARIA/touch-target exception to the 8-point scale — the visual button may be smaller but the hit area must be at least 44x44px.
- Admin action buttons: **min-height 64px** (`.bsk-btn--admin-action`) — enlarged for touchscreen fat-finger safety.
- Card inner gap: **16px** for both `.bsk-card` and `.bsk-card--admin` — unified to the nearest standard value; prior 18px gap consolidated to 16px.

### New Components (Phase 08 adds)

#### 1. Admin Menu Close Button (`#admin-btn-close`)

- **Position:** Top-right corner of `.bsk-card--admin`, absolutely positioned relative to the card.
- **Visual:** Text character `×` (`&times;`). Font size 20px, weight 700, color `#FFFFFF`.
- **Style:** Uses `.bsk-btn` base class. No yellow border. Background `#2A2A2A`, border `#3A3A3A`. Discreet — not primary action appearance.
- **Size:** `min-width: 44px; min-height: 44px` (touch target compliance, D-01).
- **Active state:** `background: #3A3A3A` (standard `.bsk-btn:active`).
- **Placement in DOM:** First child of `.bsk-card--admin`, absolutely positioned `top: 8px; right: 8px` relative to the card (card gets `position: relative`).
- **aria-label:** `"Admin-Menü schließen"`
- **Keyboard:** Esc key routes through same `admin:close` handler — no additional visual representation needed.

#### 2. "PIN ändern" Admin Button (`#admin-btn-pin-change`)

- **Style:** `.bsk-btn .bsk-btn--admin-action` — identical to all other admin action buttons.
- **Label:** `PIN &auml;ndern`
- **Position in button stack:** After `#admin-btn-credentials` ("Anmeldedaten ändern"), before `#admin-btn-update-config` ("Auto-Update einrichten"). Follows Phase 5 D-02 safe-to-destructive ordering.

#### 3. PIN Change Overlay (`#pin-change-overlay`)

- **Layer:** z-index 400 — same layer as credentials overlay and PIN modal (mutually exclusive with both).
- **CSS class:** `.bsk-layer .bsk-layer--credentials` — reuses the same backdrop class (semi-opaque `rgba(26, 26, 26, 0.97)`).
- **Card:** `.bsk-card` (not `.bsk-card--admin`) — standard 420px card, `border: 2px solid #F5C518`.
- **Title:** `PIN &auml;ndern` using `.bsk-card-title`.
- **Fields (in order):**
  1. "Aktuelle PIN" — `type="password"`, `inputmode="numeric"`, `pattern="[0-9]*"`, `minlength="4"`, `maxlength="6"`, `autocomplete="off"`
  2. "Neue PIN" — same attributes
  3. "PIN best&auml;tigen" — same attributes
- **Error area:** `.bsk-field-error` `id="pin-change-error"` — single error element below the third field, hidden by default.
- **Action buttons (in order):**
  1. "Speichern" — `.bsk-btn .bsk-btn--primary` (yellow, disabled until all fields valid)
  2. "Abbrechen" — `.bsk-btn` (neutral grey, returns to admin menu)
- **Note on implementation choice (Claude's Discretion, CONTEXT.md):** Use a new `#pin-change-overlay` `<div>` rather than a third mode on `#credentials-overlay`. Rationale: keeping a separate div avoids three-way mode logic in `showCredentialsOverlay()` and keeps `#credentials-overlay` semantics clean (Magicline credentials only). HTML is simpler; host.js handler is simpler.

## Interaction Contract

### Close Button Behavior (ADMIN-01)

| Trigger | Condition | Outcome |
|---------|-----------|---------|
| Click `#admin-btn-close` | Admin menu visible, no nested overlay open | `admin:close` → hide admin menu, show prior layer |
| Esc keydown | Admin menu visible, no nested overlay open | Same as click |
| Ctrl+Shift+F12 (second press) | `adminMenuOpen === true` | Same as click — skip PIN modal |
| Click `#admin-btn-close` | Admin menu visible, PAT lockout active | Close panel WITHOUT resetting lockout countdown |
| Esc keydown | Credentials overlay visible | Credentials overlay "Abbrechen" path — does NOT cascade to admin close |
| Esc keydown | PIN change overlay visible | PIN change overlay "Abbrechen" path — does NOT cascade to admin close |

**Prior layer restoration:**
- If Magicline view exists: show cash register (`setMagiclineViewVisible(true)`)
- If no Magicline view (welcome state): send `welcome:show`

### Credentials Re-Entry Mode (ADMIN-03)

| Mode | Trigger | Fields Shown |
|------|---------|-------------|
| `first-run` | Cold boot, first-run flag not set | All 4 fields: PIN, PIN confirm, username, password |
| `re-entry` | Admin clicks "Anmeldedaten ändern" | Username + password fields ONLY; `#creds-firstrun-fields` `display:none` |

Card title changes by mode:
- `first-run`: "Kiosk einrichten" (existing)
- `re-entry`: "Anmeldedaten &auml;ndern"

### PIN Change Flow (D-09, D-10, D-11)

```
"PIN andern" button click
  → hide admin menu (send hide-admin-menu IPC)
  → show #pin-change-overlay
  → user fills "Aktuelle PIN", "Neue PIN", "PIN bestatigen"
  → "Speichern" enabled only when all 3 fields non-empty and lengths 4–6
  → on submit:
      [1] verifyPin(store, currentPin)
          → fail: show "Falscher PIN" in #pin-change-error, clear fields
          → pass: continue
      [2] newPin === confirmPin
          → fail: show "PINs stimmen nicht uberein" in #pin-change-error
          → pass: continue
      [3] newPin.length >= 4
          → fail: show "PIN muss mindestens 4 Ziffern haben" in #pin-change-error
          → pass: setPin(store, newPin), emit audit log, return to admin menu
  → "Abbrechen": hide #pin-change-overlay, show admin menu
```

### Reload from Welcome State (FIX-01)

| State | "Kasse nachladen" tap | Outcome |
|-------|-----------------------|---------|
| Magicline view exists | `magiclineView.exists() === true` | Existing: `magiclineView.reload()` + authFlow restart |
| Welcome state | `magiclineView.exists() === false` | Close admin → trigger fresh welcome-tap session start → splash appears |
