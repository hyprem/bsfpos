<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| USB HID reader → main process | HID keystrokes arrive via before-input-event from untrusted physical source (anyone holding a badge or a USB keyboard) |
| main process → Magicline page | Badge content is injected via executeJavaScript — string interpolation is a tampering surface |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | badgeInput.commitBuffer executeJavaScript | mitigate | Use `JSON.stringify(committed)` for the value interpolation — escapes quotes, backslashes, and control chars. No raw string concat of buffer content. Verified by test case 'committed payload is JSON.stringify escaped'. |
| T-04-02 | Tampering | badgeInput buffer length/content | mitigate | `input.key.length === 1` filter rejects multi-char keys; `buffer.length > 3` minimum blocks stray single-char injections; Enter/Tab terminators bound buffer lifetime. ASVS V5.1.1. |
| T-04-03 | Information Disclosure | badgeInput log lines | mitigate | log.info line logs `length=N` only, never committed content. `! grep "log.info.*committed"` acceptance check enforces. Phase 5 ADMIN-05 alignment. |
| T-04-04 | Denial of Service | idleTimer timer flood from bump() | accept | bump() is a single clearTimeout+setTimeout — O(1) and cheap. High-frequency bumps (e.g. rAF activity from inject.js) cannot exhaust resources. |
| T-04-05 | Tampering | Race: executeJavaScript fires against destroyed wc after sessionReset | mitigate | `if (wc.isDestroyed()) return;` guard before executeJavaScript (Pitfall 7). Unit tested. |
| T-04-06 | Repudiation | State transitions without audit trail | mitigate | Every idleTimer state transition emits `idleTimer.state: X -> Y reason=Z` via electron-log. Grep-verifiable in acceptance criteria. |
</threat_model>
