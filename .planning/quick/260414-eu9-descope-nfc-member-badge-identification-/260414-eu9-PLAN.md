# Quick Task 260414-eu9 — Descope NFC member-badge identification from v1.0

## Why

Magicline staff account would need member-lookup permissions to translate badge IDs to members. The user does not want to grant those permissions to the kiosk's staff account. A card terminal next to the kiosk handles all payment via Magicline's "Jetzt verkaufen" → "Kartenzahlung" flow, so member identification at the kiosk is no longer required for v1.0. NFC member identification is permanently dropped from v1.0 (may return in a later milestone).

## Scope (locked)

### Code removal

1. **Delete `src/main/badgeInput.js`** — HID buffering + arbitration module.
2. **`src/main/main.js`** — remove `badgeInput.start()`, remove the `before-input-event` capture wiring tied to badge input. Add a one-line focus call: after `cash-register-ready` fires (the existing splash-hide trigger), execute JS in the magicline view to focus `[data-role="product-search"] input` so HID wedge keystrokes land there naturally.
3. **`src/inject/inject.js`** — remove the customer-search React-value-setter injection path (`fillCustomerSearch` / `injectBadgeIntoCustomerSearch` / similar). Keep all other injection logic untouched.
4. **`src/inject/fragile-selectors.js`** — drop `CUSTOMER_SEARCH_*` selector entries. Keep `PRODUCT_SEARCH_*` and `JETZT_VERKAUFEN_*`.
5. **`src/inject/inject.css`** — KEEP the customer-search hide rule. Members shouldn't see the field even though we no longer use it.
