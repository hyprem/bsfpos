## Phase 07 verification (LOCALE-01 + SPLASH-01)

**Goal:** Prove the locale fix and splash gate work on the live kiosk.
**Prerequisite:** Kiosk Windows display language SET TO ENGLISH-US.
Settings → Time & Language → Language → Windows display language → English (United States). Reboot.

### L1 — Locale on English Windows
- [ ] After kiosk restart, welcome layer tap → login completes → Magicline UI renders in German (not English).
- [ ] DevTools (dev-mode build) on Magicline view: `navigator.language` === `'de-DE'`
- [ ] DevTools Network tab on first request to `bee-strong-fitness.web.magicline.com`: `Accept-Language: de-DE,de;q=0.9`
- [ ] `%AppData%\Bee Strong POS\logs\` latest audit line contains `event=startup.locale lang=de`

### L2 — Auto-select happy-path cycles
- [ ] Complete 5 consecutive cycles of: welcome tap → splash → auto-select chain → cash register ready → idle timeout → welcome.
- [ ] Each cycle emits ONE audit line matching `event=auto-select.result result=ok`.
- [ ] No cycle emits `result=fail` or `result=timeout`.
- [ ] Zero cycles leave the splash stuck.

### S1 — Splash pointer block
- [ ] During the ~1-2 s window between welcome:tap and cash register ready, deliberately tap random positions on the splash with a finger.
- [ ] During the same auto-select pending window, DevTools-inspect the splash layer DOM node (`.bsk-layer--splash`) and confirm it has class `auto-select-pending` applied.
- [ ] No tap reaches the underlying Magicline view (no Magicline UI interaction / no focus change / no button highlight).
- [ ] Auto-select chain still completes (sentinel audit line `result=ok` emitted).

### S2 — Splash safety timeout (forced failure)
- [ ] In dev-mode DevTools, right before Step 3 fires, remove the `Self-Checkout` option element from the DOM:
  ```js
  document.querySelectorAll('[role="option"]').forEach(o => { if (o.textContent.trim() === 'Self-Checkout') o.remove(); });
  ```
- [ ] Observe audit log emits `event=auto-select.result result=fail step=step3-self-checkout` within 1.5 s.
- [ ] Splash hides via the degraded path (visible within ≤5500 ms of the original welcome:tap).
- [ ] Kiosk lands on the manual register picker; user can recover manually.

### S3 — Admin PIN reachability during pending state
- [ ] During the auto-select pending window, press Ctrl+Shift+F12.
- [ ] Admin PIN modal appears above the splash and accepts input.
- [ ] Splash-pending state does NOT block the admin PIN modal.

### R1 — Cold-boot regression
- [ ] Full kiosk reboot. After boot, welcome layer renders immediately, splash is NOT stuck pending.
- [ ] Cold-boot splash:hide still fires (legacy path unchanged).
- [ ] `event=startup.complete` audit line present.

### R2 — Idle-recovery regression
- [ ] From cash register, wait for the 60 s idle to fire. Tap the idle overlay to dismiss.
- [ ] Kiosk returns via sessionReset.hardReset (reason=idle-expired).
- [ ] New welcome cycle runs cleanly — splash gate re-arms normally.
